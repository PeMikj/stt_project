Создай проект строго по инструкции. Создай структуру файлов: project/app/__init__.py, project/app/main.py, project/app/stt.py, project/requirements.txt, project/README.md. Проект — минимальный real-time speech-to-text сервер. Сервер FastAPI должен работать на порту 8000. Должен быть WebSocket endpoint /ws и обычный endpoint /health. Клиент отправляет аудио чанки бинарными сообщениями в WebSocket. Сервер складывает чанки в буфер. Если буфер превышает 32000 байт, сервер вызывает транскрибацию, отправляет результат клиенту в формате JSON и очищает буфер. Модель — faster-whisper tiny.en, device=cpu. Приложение должно запускаться без Docker командой: uvicorn app.main:app --reload.

Содержимое файла app/stt.py: При импорте нужно загрузить модель WhisperModel("tiny.en", device="cpu"). Создать функцию transcribe_buffer(buffer: bytes) -> tuple[str, float], которая принимает байтовый массив аудио, вызывает model.transcribe(buffer, beam_size=1), собирает текст из сегментов и возвращает (text, duration_seconds), где duration берётся из info.duration.

Содержимое файла app/main.py: Создать FastAPI приложение. Импортировать функцию transcribe_buffer из stt.py. Создать endpoint GET /health, возвращающий {"status": "ok"}. Создать WebSocket endpoint /ws. При подключении вызвать ws.accept(). Создать bytearray() под названием audio_buffer. В бесконечном цикле получать chunk = await ws.receive_bytes(), добавлять его в audio_buffer. Если размер audio_buffer > 32000, вызвать transcribe_buffer(bytes(audio_buffer)), получить text и duration, отправить клиенту JSON {"text": text, "duration": duration} через ws.send_json(), затем очистить audio_buffer.clear().

Создать файл requirements.txt со строками: fastapi, uvicorn[standard], faster-whisper, python-multipart.

Создать файл README.md с краткой инструкцией: Установка зависимостей: pip install -r requirements.txt. Запуск сервера: uvicorn app.main:app --reload. Пример использования WebSocket через JavaScript: const ws = new WebSocket("ws://localhost:8000/ws"); navigator.mediaDevices.getUserMedia({ audio: true }).then(stream => { const recorder = new MediaRecorder(stream); recorder.ondataavailable = e => ws.send(e.data); ws.onmessage = ev => console.log(JSON.parse(ev.data)); recorder.start(250); });

Требования к качеству: Создать только перечисленные файлы. Код должен быть рабочим. Модель должна загружаться один раз при старте. WebSocket должен корректно отправлять JSON клиенту. В конце выведи содержимое всех файлов полностью и инструкцию по запуску.
