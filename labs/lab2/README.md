## Сервисы в проекте

### 1. `init_redis` — одноразовый init-контейнер
- Ждёт Redis перед запуском приложения  
- Выполняется один раз и завершается  

### 2. `redis` — база данных
- Работает как in-memory key-value storage  
- Сохраняет данные в volume  
- Имеет `healthcheck`  

### 3. `stt_app` — основное приложение
- Автоматически собирается из `Dockerfile`  
- Зависит от `init_redis` и `redis`  
- Имеет проброшенный наружу порт  
- Использует переменные окружения из `.env`  
- Подключён к `redis` через internal network  


## Ответы на вопросы

### 1. Ограничение ресурсов
- Ограничивать ресурсы **можно**
- Корректный способ — через Docker Swarm (`deploy.resources`)
- В обычном `docker-compose` работают только:
  - `mem_limit`
  - `cpus`

### 2. Запуск отдельных сервисов
- Можно запускать сервисы выборочно  
- Команда:
  ```bash
  docker compose up <service_name>
 ```
